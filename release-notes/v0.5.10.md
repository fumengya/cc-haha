# v0.5.10

> 本版本聚焦"新会话从上一个会话继续"体验,补全 v0.5.9 接续功能的最后一公里;同时引入插件依赖自动检测 + 一键安装,以及一批桌面端 UX / 可访问性改进。

## 新功能

- **会话接续:深度模式 toggle**(#13)。新建会话欢迎页"最近活动"卡片新增"深度"开关:打开后,下次点击"从这里继续"会把 verbatim 对话尾部从默认 ~4k tokens 拉到 ~12.5k tokens(60 轮 / 每轮 800 字 / 50k 总额),让新会话 AI 能看到完整的"最近 debug 转折"。开关状态持久化到 `localStorage`,选了一次不用每次重选。
  - 服务端零额外 LLM 成本:复用已缓存的 LLM 摘要 `main` / `recent`,只重新切片 verbatim 尾部。
  - 默认模式不变(还是 v0.5.10 的 24/600/16k),深度模式硬编码不受 env override 影响,UX 可预期。
- **会话接续:默认 verbatim 尾部翻倍**(#12)。`SessionSummary.recentRaw` 的默认尺寸从 v0.5.9 的 12 轮 / 400 字 / 8000 字符总额提升到 **24 轮 / 600 字 / 16000 字符总额**(≈ 4k tokens),覆盖一整个 debug 转折的前因后果。env override 上限同步从 50/30k 拉到 80/50k。
  - 用户反馈:v0.5.9 的尾部只覆盖 ~6 个 user-AI 对,接续后 AI 经常不知道"刚才到底卡在哪",新尺寸覆盖 ~12 对,实测够用。
- **插件依赖自动检测 + 弹窗**(#8)。开启 MCP 插件时,服务端用 `where` / `command -v` 探测每个 stdio server 声明的 host 命令是否在 PATH,缺啥直接弹"缺少依赖"模态,带文档链接、可复制安装命令、affected servers 列表。检测纯只读,不 spawn 工具本体;结果 60s 缓存避免一次开 7 个 server 触发 7 次重复探测。
  - 配套 schema 扩展(`McpStdioServerConfigSchema.prerequisites`)、`prerequisitesService`、`GET /api/plugins/prerequisites`、`PluginPrerequisitesModal`。
- **一键安装缺失依赖**(#9)。`PluginPrerequisitesModal` 底部新增"一键安装(N)"按钮:打开终端 tab → 等待终端就绪 → 按 150ms 间隔逐条注入每行依赖的首选安装命令(winget / scoop / brew / apt / dnf 看平台)。Watched automation 而非静默执行,用户全程能看到、能 Ctrl-C 打断。
- **逆向插件依赖声明**(#10)。`reverse-engineering` 插件的 7 个 stdio server(ghidra / radare2 / gdb / lldb / jadx / apktool / frida)全部声明 `prerequisites[]`,弹窗去重后呈现 5 个独立行:`uvx` / `npx` / `radare2` / `gdb` / `java`,各自带 win32 / darwin / linux 的安装命令。
- **桌面 UX Phase 1 + Sidebar 重构**(#11)。
  - **可访问性**:`MessageList` 加 `role="log"` + `aria-live="polite"`,`StreamingIndicator` 加 `role="status"`,`TabBar` 滚动按钮加 `aria-label`,浏览器地址栏 / 浏览面板硬编码中文字符串改走 i18n。
  - **新共享组件**:`Skeleton`(text / rect / circle 三种变体,带 `role="status"` + `aria-label="Loading"`)和 `Tooltip`(DOM 实现,支持快捷键提示槽)。
  - **侧边栏骨架占位**:`Sidebar` 初始拉取阶段不再显示 "Loading..." 文本,改用 6 行 skeleton 网格;`TraceListSkeleton` 也迁移到共享组件。
  - **Sidebar 重构**:`Sidebar.tsx` 从 1979 行精简到 ~1150 行,抽出 `sidebarUtils.ts`(项目分组 / localStorage 持久化 / 偏好 / 路径处理 / 时间格式化)和 `sidebarComponents.tsx`(图标 / `NavItem` / `ProjectHeaderMenu` / `SessionRowMeta` 等)。纯结构拆分,行为不变。
