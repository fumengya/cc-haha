# v0.5.27

> 这一版聚焦两个高频工作流细节和一组诊断包追出的稳定性修复：image-gen 插件支持把生成/编辑出的图片直接保存到用户指定目录，避免 base64 图片长期堆在对话上下文里；侧边栏会话右键菜单补上“复制文件路径”和“在文件管理器中打开”，方便把 `.jsonl` 转录交给其它工具继续处理；同时修掉 bundled sidecar 图像处理误报缺 `sharp`、task output symlink 失败后误删不存在文件的问题。

## 新增

- **image-gen 支持 `output_dir` 保存图片到本地目录**。`generate_image` 与 `edit_image` 新增可选参数 `output_dir`：provider 返回 `b64_json` 时插件会将 base64 解码后写入该目录；provider 返回 URL 时会先经过现有 URL 安全校验，再下载图片并保存。保存成功后 tool result 返回 `file://` markdown 图片链接和 `Saved to: ...` 路径，未传 `output_dir` 时保持原有行为——base64 仍作为 MCP image content 返回，URL 仍作为 markdown 远程图片返回。
- **会话右键菜单新增“复制文件路径”和“在文件管理器中打开”**。桌面端会话列表可以快速复制 `.jsonl` 绝对路径，或直接在系统文件管理器中定位该文件，方便转交给其它 AI、编辑器或归档工具。
- **图片附件支持基础标注**。在输入框图片缩略图上点击标注按钮，可以用框选、箭头和文字对图片做快速说明；保存后会替换为带标注的图片再发送给模型。

## 修复

- **避免 image-gen base64 图片污染长期对话上下文**。对支持 `b64_json` 的模型（例如 `gpt-image-2` / `gpt-image-1` / Gemini image）使用 `output_dir` 后，返回内容切换为本地文件链接，不再把完整 base64 图片塞进对话结果。
- **修复 bundled sidecar 图像处理误报缺 `sharp`**。打包后的 sidecar 本应优先使用内置 `image-processor-napi`；当 native processor 不可用时，不再继续尝试加载外部 optional `sharp` 包，避免诊断中出现 `Cannot find package 'sharp' from claude-sidecar.js` 这类误导性错误。`FileReadTool` 的最后一层图片压缩兜底也统一走 `getImageProcessor()`。
- **修复 task output symlink 失败后的 ENOENT 噪声**。`initTaskOutputAsSymlink` 现在只在 `EEXIST` 时删除旧 output path 并重试；权限/平台策略等其它 symlink 错误会直接走 fallback，不再无条件 `unlink(...tasks/*.output)` 导致 `ENOENT` 进入诊断包。
