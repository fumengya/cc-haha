# v0.5.21

> 合并上游 NanmiCoder/cc-haha v0.4.3：吃到 mid-stream retry / 1M provider context / subagent 流式输出三处 bug 修复，融合 Cmd+K 全局会话搜索、模型 1M context 复选框、WorkspacePanel Navigator 切换三处新 UI；保留 cc-haha 这边 #71-#79 的 skills / project rules / agent-limiter / chat 重试去噪。

## 修复（来自上游）

- **mid-stream 流式重试**。`withRetry` 现在会识别 `api_error` / `overloaded_error` 这类没有 HTTP status 的中流错误，按 `CLAUDE_STREAM_TRANSIENT_RETRY_MAX`（默认 2）重试，避免一次工具调用半途出错就直接打断会话。配套新增 `RetriableStreamError` / `isRetryableStreamError` / `getMaxStreamTransientRetries`。
- **provider 1M context 支持**。`SavedProvider` 新增 `model1mSupport: { main, haiku, sonnet, opus }`。打开后 runtime 会把模型 ID 转成 `<model>[1m]` 形式，与上游约定的 1M context window 协议对齐。Settings 页每个模型槽下方多出一个 1M 复选框。
- **后台子代理工具活动流式回灌 desktop**。`extractAgentToolActivities(message)` 把子代理消息里的 `tool_use` / `tool_result` 块抽出来，桌面端能在子代理跑动期间看到它的工具调用进度而不是只等最终结论。

## 新功能 / UI（融合上游）

- **Cmd+K 全局会话搜索**。Sidebar 在本地过滤输入框上方加了一颗带 ⌘K 标的 “Search chats” 按钮，点开 `GlobalSearchModal`；`useKeyboardShortcuts` 把 Cmd+K 也桥接到这个 modal。我方原本的局部 filter input 行为完全不动，两个能力并存。
- **WorkspacePanel Navigator 切换**。Tab 栏右侧多出 `account_tree / right_panel_close` 切换按钮；同时把 tab 改成上游的嵌套 button 结构（`role="tab"` + `aria-selected`），可访问性更好。脏标记 `●` 仍然保留。
- **空 session hero 紧凑模式**。当底部 docked terminal 可见（`compactEmptyHero`）时，欢迎页内边距从 `pb-32` 收缩到 `pb-6`，避免被终端面板挡住。
