# v0.5.31

> 这一版修复桌面端后台任务状态与停止控制问题，覆盖运行中任务无法取消、Agent 完成后主对话不继续输出、后台任务长时间占用资源等场景。

## 重点更新

- **后台任务可手动停止**：后台任务抽屉为运行中任务新增“停止”按钮，桌面端会通过 `stop_background_task` 转发到 CLI 的 `stop_task` 控制通道。
- **Agent 完成后主对话继续输出**：修复 Agent / 子代理完成通知被当作普通后台任务终止通知处理，导致后续 assistant 内容被静默吞掉的问题。
- **运行状态更准确**：停止任务后前端会乐观标记为 `stopped`，并根据剩余运行中任务更新顶部运行状态，避免“已停止但仍显示运行中”的误导。
- **后台资源回收链路打通**：停止请求复用现有 `stopTask()`，覆盖 local bash、local agent、remote agent 的 kill / abort / cleanup / output eviction 路径。
