# v0.5.26

> 这一版的主线只有一件：H5 公网访问从"手动接 ngrok / 反代"升级成"设置页一键 Cloudflare 隧道"——cloudflared 由桌面端管理，公网 URL 自动回填成 H5 的 effective publicBaseUrl，QR 码/连接绑定/CORS 跟着走，不再需要手填 `CLAUDE_H5_TUNNEL_URL`。顺带补了一个停启隧道时 URL 残留导致手机 1033 的真 bug，并把左侧会话列表的右键菜单补全成可以快速把 `.jsonl` 转录交给别的 AI。

## 新增

- **一键 Cloudflare 公网隧道**（#945）。设置 → H5 访问页底部新增"一键公网隧道（Cloudflare）"区块：选模式（快速/命名）→ 命名模式贴 token → 点"启动隧道"，桌面主进程 spawn `cloudflared` 子进程，从 stderr 抓出 `https://*.trycloudflare.com` 或用绑定域名，调 server 的 `tunnel/report` 端点把 URL 注入到 `h5AccessService` 的进程内 runtime 覆盖位（最高优先级，胜过 `CLAUDE_H5_TUNNEL_URL` 与 `CLAUDE_H5_PUBLIC_BASE_URL` 两个环境变量），随后 settings/QR/CORS 立即生效。隧道 URL 永不写盘（quick 模式每次重启都换地址，写盘会留幽灵），但 named 模式的 Cloudflare token 持久化到 `~/.claude/cc-haha/settings.json` 的 `h5Access.tunnelToken`，桌面重启后自动恢复——token 走 loopback-only 的 `/api/h5-access` GET，远程浏览器 403 拿不到。
- **会话右键菜单新增"复制文件路径"和"在文件管理器中打开"**。`SessionListItem` 加可选字段 `filePath: string`（绝对路径到 `.jsonl`），sidebar 右键菜单把它复制到剪贴板或调 Electron `shell.showItemInFolder` 在系统文件管理器里定位选中。"在文件管理器中打开"按钮通过 `desktopHost.shell.showItemInFolder` feature detection 决定是否渲染——浏览器 H5 session 不显示。新增 IPC channel `desktop:shell:show-item-in-folder`，validator 与现有 shellOpenPath 同款（字符串型 payload）。

## 修复

- **隧道 stop → start 后 UI 显示旧 trycloudflare URL，手机扫码 1033**。根因是 `desktop/electron/services/serverRuntime.ts` 的 `stopTunnel` 走 `tunnel/report { url: null }` 上报 idle，但 server 端 handler 把 `body.url` 不是字符串时映射成 `undefined`（这是上一轮修"心跳报告不能清空 URL"时刻意引入的保护，JSON `null` 也被一起当成"不动"），于是 server 端 runtime URL 永不被 report 清空——上次的 quick 隧道地址残留下来，QR 跟着挂掉。本版改 `stopTunnel` 调专门的 `POST /api/h5-access/tunnel/clear` 端点（这个端点本来就是为清空运行时 URL 设计的），与"心跳报告 status-only / url 缺失 / 空串都不清空"的契约不冲突。

## 升级注意

- 隧道 token 是明文落盘（`h5Access.tunnelToken` 在 settings.json 里），但只在 loopback-only 端点可读；远程浏览器/扫码的 H5 端永远拿不到。如果你的 `~/.claude/cc-haha/settings.json` 同步到了不受信任的位置，要按这一点重新评估。
- quick 模式每次启动都会拿到一个新的随机 trycloudflare 域名；如果想保持长期固定地址，请用 named 模式 + 你绑到 Cloudflare 的子域名 + Tunnel token。
- 一键隧道只在 Electron 桌面端可用（spawn cloudflared 在主进程）；浏览器 H5 session 不显示该按钮——继续支持手动设 `CLAUDE_H5_TUNNEL_URL` 环境变量。
- 会话右键菜单的"在文件管理器中打开"只在桌面端可见（feature detection），浏览器 H5 session 看不到。"复制文件路径"对老 server 返回不带 `filePath` 的会话会显示"文件路径不可用"，不会崩。
