# v0.5.24

> 这一版补两件事：把 Matt Pocock 的三个社区技能内置进**推荐技能**列表（配中文介绍），以及修一个 stop 时丢失半截流式回答的真 bug——它是早前一次不完整 revert 留下的退化。

## 新功能

- **推荐技能新增 Matt Pocock 三件套**（#86）。设置 → 技能 → 推荐技能里新增三个开箱即用的社区技能，均配中文原生介绍：
  - **mattpocock-tdd**（工程）：测试驱动开发工作流，先写失败测试再实现。
  - **mattpocock-diagnosing-bugs**（工程）：系统化定位 bug，先复现、再二分、最后修。
  - **mattpocock-grilling**（效率）：把模糊需求"逼问"清楚再动手，减少返工。
  - 安装名加 `mattpocock-` 前缀避免与现有技能冲突；SKILL.md 内的 upstream `name:` 保持不变。
  - 五个语言（en / zh / zh-TW / jp / kr）的介绍与新增分类（Engineering / Productivity / Workflow）全部齐备。

## 修复

- **Stop 时不再丢失半截流式回答**（#87）。早前回滚 pending-tool UI（`#703`）时把 `stopGeneration` 简化成了裸 `consumePendingDelta(sessionId)`，把最后一次节流 flush 到点击 Stop 之间缓冲的 `content_delta` 直接丢弃了——结果点 Stop 会吞掉模型刚吐出、还没刷进界面的那段文本。现在这段 delta 会被追加进 `streamingText`（按 sessionId 限定，不影响其它并发会话），停下来时看到的就是完整的已生成内容。
- **清理两次不完整 revert 遗留的测试退化**（#87）。`34aa9d71`（回滚 ToolSearch / pending-tool UI）和 `a6a2f661`（隐藏 ToolSearch 开关）删了功能代码却漏改对应测试，导致 main 上三个 desktop 测试常红。本版把孤儿断言对齐到现网真实行为，desktop 测试套件恢复全绿。
