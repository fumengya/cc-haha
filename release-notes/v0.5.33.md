# v0.5.33

> 这一版修复桌面聊天在 AI 持续生成时不会自动跟随最新内容的问题，并补强 PR 覆盖率门禁与任务列表测试稳定性。

## 重点更新

- **聊天输出默认持续跟随最新**：AI 消息继续增长、布局高度变化或内容流式追加时，聊天列表会保持 pinned 到底部，不再静默停留在旧位置。
- **只在用户主动回看时暂停自动滚动**：自动跟随状态现在区分“用户向上滚动”和“内容变高导致离底”，只有明确上滚查看历史时才显示 `Latest` 并暂停跟随。
- **轻度回看自动恢复**：用户轻度向上查看附近内容后，如果 5 秒内没有继续操作，会自动恢复到最新消息；深度回看仍会保持暂停，避免打断阅读。
- **切换会话更稳定**：切换 session 时会清理待执行的轻度回看恢复计时器，避免旧会话的滚动恢复影响新会话。
- **PR 覆盖率门禁更准确**：coverage gate 会识别 GitHub pull request 的 synthetic merge commit，不再把它误判为真实 merge 而跳过 changed-line coverage 检查。
- **任务列表测试更稳定**：TaskService 的重复读取回归测试不再依赖文件系统目录遍历顺序，并将读文件计数限定在任务 JSON 文件上。
