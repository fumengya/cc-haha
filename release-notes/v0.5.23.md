# v0.5.23

> 重点是**对话流式稳定性**：默认开启一次中流重试 + 用户层错误信息保证唯一。配套修一批 v0.5.21 之后累积的桌面端体验问题（批量导出、Solo Council standby、图片粘贴过大、JSONL 重复全量解析导致 endpoint timeout）。
>
> v0.5.22 是平行 local-rebuild 线的 tag，未合主线；本版直接从 v0.5.21 之后累积发布。

## 修复

- **流式中流错误自动重试一次**（#84）。`getMaxStreamTransientRetries()` 默认值从 `0` 改 `1`。第三方 relay 的 `terminated` / `fetch failed` / 上游 transient `api_error` 之前会一次抖动就直接终结对话，现在默认会无声重试一次，绝大多数瞬时故障被自然吞掉。Override 仍走 `CLAUDE_STREAM_TRANSIENT_RETRY_MAX`（设 `0` 显式禁用）。
- **重试期间用户只看到一条错误**（#85，回归保护）。把「retry 透明 + exhausted 唯一」的 UX 契约硬编进 `streamRetry.test.ts`：
  - retry 成功：UI 看到 `0` 条 error message
  - retry 全失败：恰好 `1` 条 error message
  - 防未来某次重构把每次 attempt 的错误都泄露给前端导致红色错误条堆叠
- **Solo Council 卡片不再误显 standby**（#55）。在 agent 完成前不再瞬间闪一下 standby 状态。
- **批量导出会话**（#81 + 后续修复）：
  - 项目右键菜单新增「批量导出」入口
  - 导出过程显示 loading 状态
  - 多会话打成 zip 下载
- **Settings 隐藏 ToolSearch toggle**。回滚上游 `#823 ToolSearch forced enable`，避免对不支持 web_search 的 provider 默认启用导致请求 reject。同时回滚 `#703/#707/#712/#751` 的 pending tool UI。

## 新功能

- **图片粘贴/拖入自动压缩**。粘贴或拖入聊天框的图片会被压成最长边 1568px、JPEG quality 0.85，再上传。彻底解决「贴张全屏截图就上下文超限」的体验断点；视觉无可感知差异。

## 性能

- **JSONL 读缓存**（#83）。会话 transcript 是 append-only JSONL，能涨到几十 MB。`readJsonlFile` 之前每次都全文件 `fs.readFile` + 逐行 `JSON.parse`，零缓存；打开会话一次同时跑 `getSessionMessages + getSessionTaskNotifications` 会**两次解析同一文件**。新增按 `(filePath, mtimeMs, size)` 的 LRU 解析缓存：文件未变 → 直接命中缓存。这是用户在诊断包里看到的 `Request timed out after 120s` / 主线程 starve 的主要根因之一。

## 升级注意

- 用户用第三方 relay 时，第一次抖动**不会**再终结对话；如果 relay 真的稳定挂掉，第二次失败仍然会立刻显示同样的错误消息（不会比 v0.5.21 慢任何一秒）。
- 设了 `CLAUDE_STREAM_TRANSIENT_RETRY_MAX=0` 的环境继续禁用重试，行为不变。
- 批量导出新增的 zip 流程在第一次使用时会显示 loading；不影响单会话单文件下载的旧路径。
