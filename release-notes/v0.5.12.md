# v0.5.12

> 本版本聚焦 Workspace LSP 的产品链路收尾：桌面工作区编辑器会把打开、修改、保存事件同步给 Workspace LSP 服务，并通过标准 `textDocument/publishDiagnostics` 接收语言服务器诊断。自定义 LSP 服务器配置现在可以从桌面设置持久化进入工作区 LSP 流程，并且在扩展名重叠时优先于内置预设。

## 新功能

### Workspace LSP 自动诊断链

- 工作区编辑器在文件 `open` / `change` / `save` 时调用 session LSP sync API，服务端启动匹配的语言服务器并发送 LSP 文档同步通知。
- 诊断只来自语言服务器的 `textDocument/publishDiagnostics` 通知，桌面按文件 URI 缓存并展示错误、警告、信息和提示级别诊断。
- LSP 状态接口返回当前文件匹配到的 server name、command、ready / starting / unavailable 状态和启动错误，保留工作区路径校验，拒绝越界路径。

### 自定义 LSP 支持

- 支持用户提供自定义 LSP `path` 或裸 `command`，并使用独立 `args: string[]` 传参；不接受 shell 字符串拼接，Windows 含空格的绝对路径可作为 `path` 使用。
- 自定义 `extensionToLanguage` 映射会覆盖重叠扩展名的内置预设。例如用户为 `.ts` 配置自定义 server 时，`.ts` 会路由到自定义 server，而 `.tsx` / `.js` / `.jsx` 仍可继续使用 TypeScript 预设。
- 桌面设置 store 新增 `workspaceLsp` 持久化字段，Workspace Panel 的 LSP state / diagnostics / sync 请求会自动带上该配置并触发必要的重启。

## 修复

- 修复自定义 LSP server 被内置 preset 抢占同扩展名的问题。
- 修复桌面设置中的自定义 LSP 配置未进入 Workspace LSP 产品链路的问题。

## 范围说明

- 本版本没有实现语言服务器安装器，也不打包 `typescript-language-server` / `pyright` / `rust-analyzer`。
- 本版本没有手写 rust-analyzer 或 TypeScript 诊断；诊断来源仍是语言服务器发布的 `publishDiagnostics`。
- Settings UI 的完整表单仍可后续补齐；当前交付的是可持久化的设置/API/store hook 与 Workspace LSP 流程接线。

## 验证

- 服务端 Workspace LSP 单元测试覆盖：路径越界保护、自定义绝对路径与 args 数组、自定义 `.ts` 覆盖 TypeScript 预设、`publishDiagnostics` 缓存映射、open/change/save 同步。
- 桌面 store 单元测试覆盖：自定义 LSP 设置持久化与规范化、Workspace Panel 向 LSP state / diagnostics / sync 请求注入持久化配置、空配置不注入。
